# Généralités

Pour une définition de ce qu'un `acteur` peut représenter au sein du système, merci de vous réferer au site de Foundry VTT : [Actors](https://foundryvtt.com/article/actors/)

# Types

Pour le moment, le système supporte les types d'`acteurs` suivants :

## Personnage

Le principal type d'`acteur` supporté par le système est le `personnage`. Il est utilisé pour représenter une partie importante des entités d'une partie :

 - Les personnages joueurs (PJ) utilisés par les joueurs,
 - Les personnages non-joueurs (PNJ) spécifiques, utilisés par le maitre de jeu.

## Brute

Le second type d'`acteur` supporté par le système est la `brute`. Quoique proche du `personnage`, cette entité apporte quelques différences car elle est utilisée par le maitre de jeu pour gérer les personnages non-joueurs non spécifiques ou les groupes de personnages non-joueurs.

# Notes de développement

De nouveaux types d'`acteurs` seront supportés par le système dans une étape ultérieure du développement (drones, véhicules, sprites... ) une fois que la stabilité du modèle global `acteur` aura été fermement établie.
